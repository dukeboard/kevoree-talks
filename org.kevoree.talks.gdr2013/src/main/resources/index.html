μControlleurs sont de bons candidats !

  • Les AVR sont toujours les processeurs les plus vendus aujourd'hui
  • Les μC sont peu cher et produits en masse
  • faible consommation, sûre, prédictible
  • très contraints, développement complexe
  • Compromis entre performance et consommation ?

Déployer l'intelligence directement dans les μC est plus sûr, particuliement pour les règles réflexes, lissages de mesures,etc..

μKev : modèle à composant

  • Type définition/instance (quoi/où)
  • Composants avec ports (entrée/sortie) and paramètres
  • Modèle de canaux de communication entre ports (bindings)
  • Types via code (C/Processing/Java) Instanciation via modèle 
    @ComponentType
@DictionaryType({
        @DictionaryAttribute(name = "pin")
})
@Requires({
        @RequiredPort(name = "temp")
})
public class TempSensor
    extends AbstractPeriodicArduinoComponent {
    //behavior code
}
  • micrologiciel pour les types + KevScript pour les instances

Derrière le modèle...

  • Echanges de messages asynchones
  • File d'attente FIFO devant chaque port (Sémantique d'acteur)
  • Pas OS Pas de multi-threading
  • Scheduler oportuniste (Soft RT) dirigé par
    • Nombre de messages en attentes
    • Executions périodiques des instances
    0
  • Interpréteur KevScript transactionnel
  • Génération d'une couche réflexive à plat pour l'instanciation dynamique
  • Allocation de mémoire de type Token ring

Adaptation dynamique avec (μ)Kevoree

  • Niveau 1: Parametric adaptation: parameter update

  • Niveau 2: adaptation architecturelle

Protocole expérimentale

Cas d'étude : SmartBuilding
  • Ensemble de capteurs de domotique et d'alerte (par étage/bureau)
  • Reconfigurations quotidiennes de confort et de sécurité, 6 fois par jour
  • Actionneur pour utilisateur : délai de perception ~200ms
Protocole expérimental : 500 adaptations cycliques
  • Capteur embarqué sur processeur de type AVR-8 (328,2560)
  • Processeur 16 Mhz et mémoire 1-8kByte, mémoire EEPROM et SD
  • Adaptation poussée depuis le nœud d'étage(Java), via liaison série
