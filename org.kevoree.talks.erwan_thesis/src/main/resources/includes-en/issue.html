How to manage adaptation on Clouds ?

IBM frequently cites four aspects of self-management ... Ultimately, these aspects will be emergent properties of a general architecture, and distinctions will blur into a more general notion of self-maintenance. *
  • Each adaptation has impact on the running system
  • Each adaptation may have side effects
    • Side effects may have impact on the other adaptations
  • We need to coordinate adaptation between layers

*Kephart, J. O., & Chess, D. M. (2003). The vision of autonomic computing

How to coordinate adaptation ?

  • Use only one adaptation engine that manage all layers
  • Coordinate all adaptation engines
  • We need to have a common abstraction for all the adaptation engines
  • We need to represent all layers together
  • We need a global abstraction

What kind of global abstraction ?

  • Multi-layers
    • Represents Software, Platform, Infrastructure
  • Hosting
    • Represents the location of software on platforms
    • Represents the location of platforms on infrastructures
  • Distribution
    • Represents interconnection between machines
    • Represents distribution of the services
  • Heterogeneity
    • Represents any implementation of Software, platform, Infrastructure
    • Allows to define specific properties for each implementation
  • Extensibility
    • Allows to increase the layers (e.g Hardware)
    • Allows to extends hosting (e.g. Hardware hosts Infrastructure)
    • Allows to manage new implementation easily
  • Offline reflexion
    • Allows to check new configuation generated by adaptation process before applying it
