What is Model@Runtime ?

  • Abstraction layer
    • Architectural model which defines how the system is configured
      • Components and their interactions
      • Nodes
  • Offline reflexion layer
    • Dynamic adaptive system are online reflexive systems
      • Strongly connected with the running system
        • Misconfiguration can crash the system
    • Test and validate configuration before applying it

Model@Runtime process

Kevoree Concepts

  • A Component-based approach
    • ComponentType
      • Encapsulate business features
  • Extend with distributed features
    • ChannelType
      • Encapsulate communication semantics
    • NodeType
      • Encapsulate runtime heterogeneity
    • GroupType
      • Encapsulate model@runtime dissemination semantics

Kevoree Kloud
How to build and adapt Cloud system ?

What is a Cloud system ?

  • Layers are
    • Distributed systems
    • Heterogeneous systems
    • Dynamic adaptive systems (DAS)
    • Kevoree systems

How Kevoree modelize Cloud systems ?

  • Components
    • Software (SaaS)
    • Layer management (PaaS, IaaS)
  • Nodes
    • Software runtime (SaaS)
    • VM system (PaaS)
    • Concrete hardware system (IaaS)
  • All layers are independant
  • Multi-layers is only partial

Kevoree MetaModel

  • Kevoree Node
    • Hosts components
    • Hosts nodes

Kevoree MetaModel

  • A parent node has responsibility for hosting element's life cycle
    • add, start, stop, remove, ...
    • Each node can define its own adaptation capabilities

IaaS and PaaS nodes

  • Basic adaptation primitives: Add/Remove node
    • Specific implementations for each node type
  • Available node types
    • MiniCloud : Add/Remove Java Virtual Machine
    • FreeBSD Jails: Add/Remove VM in the same kernel space
    • LibVirt: Add/Remove VM with hardware virtualisation (on going work)
    • Amazon EC2: Add/Remove EC2 Virtual Machine (on going work)
