Quelles caractéristiques pour une abstraction de Cloud ?

  • Multi-niveaux
    • Représente les applications, les plates-formes, les infrastructure
  • Hosting
    • Représente l'hébergement des applications sur les plates-formes
    • Représente l'hébergement les plates-formes sur les infrastructures
  • Distribution
    • Represents interconnection between machines
    • Represents distribution of the services
  • Heterogeneity
    • Permet de représenter l'ensemble des implémentations existantes
    • Permet de faire cohabiter plusieurs implémentations
    • Permet de représenter les spécificités d'une implémentation
  • Extensibilité
    • Permet d'intégrer de nouvelles implémentations et leur spécificité
  • réflection désynchronisée
    • Permet de valider une configuration avant de l'appliquer
