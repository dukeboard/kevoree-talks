Quelles caractéristiques pour une abstraction de Cloud ?

  • Multi-niveaux
    • Pour représenter l'ensemble des éléments présents dans un Cloud
  • Hébergement
    • Pour localiser l'exécution des services
  • Distribution
    • Pour en tirer partie*
      • Dans la définition de nouvelles adaptations
      • Dans le fonctionnement des services

    *Guerraoui, R. et al - Oo distributed programming is not distributed oo programming - 1999

  • Hétérogénéité
    • Pour représenter de manière équivalente plusieurs implémentations d'un même service tout en explicitant leurs spécificités
  • Extensibilité
    • Pour pouvoir intégrer les futures implémentations
  • Réflection désynchronisée
    • Pour éviter les reconfigurations erronées
Distribution:
- Gerraoui a affirmé que pour faire des systèmes distribués efficaces, il est nécessaire d'expliciter cette notion de distribution plutôt que de la masquer

e.g. migration de composants pour rapprocher ceux qui interagissent entre eux => limiter la consommation de bande passante

Hétérogénéité:
e.g. être capable de différencier les types de virtualisation peut servir pour le choix de l'hébergement des machines virtuelles
