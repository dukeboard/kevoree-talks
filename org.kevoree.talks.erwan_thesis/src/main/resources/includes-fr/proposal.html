Thèse de François Fouquet

Qu'est-ce que le Model@Runtime ?

  • Objectif :
    • Réduire la séparation entre phase de conception et exécution d'une application
        En apportant les outils de la phase de conception à l'exécution
  • Support réflexif désynchronisée
    • Les systèmes adaptable sont basés sur le principe de la réflexion
      • Fortement connecté avec le système en cours d'exécution
        • Une erreur dans la reconfiguration peut mettre le système dans un état instable
    • Permet de tester et valider une configuration avant même de l'appliquer
These François Fouquet

Model@Runtime dans Kevoree

Une couche d'abstraction architecturale
  • Basée sur une approche à composant
    • Composant
      • Encapule la logique métier d'une application
  • Étendue avec des concepts de distribution
    • Canal de communication (channel)
      • Encapsule la sémantique de communication
      • Correspond au connecteurs*

      *PHUNG-KHAC, An et al. - A Model-driven Architecture-based Approach to Runtime Adaptable and Evolvable Distributed Collaborations - 2008

    • Nœud
      • Explicite la notion d'hébergement
      • Explicite les spécificités d'exécution
    • Groupe
      • Encapsule la sémantique de dissémination des mises à jour
  • Manipulable par un langage d'interaction tel que fscript*

    *DAVID, P.-Ch. et al. - Safe dynamic reconfigurations of fractal architectures with FScript - 2006.

Patron Type/Instance*

  • Type
    • Définition des spécificités d'une implémentation
  • Instance
    • Localisation de l'utilisation d'une implémentation
  • Permet de raisonner sur la substituabilité des fonctionnalités
  • Permet d'identifier les instances concernés par une mise à jour d'implémentation (de type)

*Johnson, R. et Woolf, B. - Type object - 1997

KevoreeKloud
Une extension de Kevoree pour modéliser un Cloud

Extension du métamodèle (1)

  • Les nœuds Kevoree hébergent
    • Des composants
    • Des nœuds

Extension du métamodèle (2)

  • Le nœud est responsable de l'adaptation des entités qu'il héberge
    • Ajout, démarrage, arrêt, suppression de composants, ...
    • Explicitation des capacités d'adaptation pour les types de nœuds
  • Le nœud est responsable de la phase de planification et d'exécution
Explicitation:
Par exemple, LibVirt est capable de dynamiquement modifié la quantité de mémoire allouée à une VM

KevoreeKloud API
Extensibilité et Réutilisation

Implémentation

  • Qu'est-ce qu'un nœud de Cloud (PaaS ou IaaS) ?
    • Un ensemble de propriétés spécifiques
    • Un ensemble de capacités d'adaptation spécifiques
    • Un ensemble d'implémentations des capacités d'adaptation
    • Une implémentation de la planification
  • Quelles similarités dans les implémentations de Cloud ?
    • Implémentations de même niveau capacités d'adaptation communes
      • PaaS : Gestion des composants et channels
      • IaaS : Gestion des nœuds de plate-forme
      • ajout, démarrage, arrêt, suppression
    • Idem pour certaines propriétés
      • RAM, CPU, OS, ...
  • Mutualisation des implémentations

Définition de type abstraits

  • Mise en commun de caractéristiques pour chaque niveau
    • Nœud hébergeant des nœuds
      • capacités d'adaptation spécifiques (ajout, suppression)
    • Nœud de proxy (accès à des Clouds non gérés par KevoreeKloud)
      • Point d'accès, identifiant(s) de sécurité
    • Nœud de Cloud (IaaS, PaaS)
      • Architecture matérielle, OS, RAM, CPU
    • Nœud d'infrastructure
      • plage réseau

Définition d'implémentations de base

  • Planification du JavaSeNode
    • Gestion des applications (composant, channel) et des groupes
  • Extension pour le Cloud
    • Intégration des capacités d'adaptation pour le Cloud
      • Ajout, démarrage suppression, arrêt d'un nœud

Héritage et Composition

  • Composition des spécifications
    • Propriétés et types d'adaptation
  • Héritage/surcharge des implémentations
    • Capacités d'adaptation et planification
  • Définition d'équivalence de type
    • Utile pour la généricité des gestionnaires d'adaptation
  • Simplicité de comparaison de type
    • parcours de type
