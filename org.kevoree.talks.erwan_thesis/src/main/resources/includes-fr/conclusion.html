Conclusion

  • Approche Model@Runtime pour l'adaptation de système de Cloud
    • Kevoree (collaboration)
      • Approche Model@Runtime pour concevoir et gérer des applications distribuées hétérogènes et adaptatifs
    • KevoreeKloud
      • Extension de Kevoree pour le Cloud
        • Permet d'avoir une représentation globale
    • Les groupes Kevoree
      • Permet de gérer la dissémination des reconfigurations au travers des niveaux
      • Permet de faire du partitionnement/filtrage de modèle
        • Pour limiter les informations fournies
        • Pour conserver l'indépendance entre les niveaux
    • KevoreeKloud API
      • Facilite la réutilisation et l'implémentation de nouveaux types
  • Permet la gestion de l'adaptation de manière transverse
    • Vision globale pour utiliser des informations des différents niveaux
    • Accès aux capacités d'adaptation de tous les niveaux
    • Possibilité de coordonner l'adaptation entre les différents niveaux
